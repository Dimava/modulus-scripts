# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Runtime C# scripts for the game *Modulus*, hot-reloaded by **ScriptEngine** (a MelonLoader mod). There is no traditional build step for scripts â€” saving a `.cs` file in `Scripts/` triggers Roslyn to compile and reload it within ~300ms while the game is running.

## Lifecycle

Every script follows this pattern:

```csharp
public static class MyScript
{
    static readonly Harmony _h = new("my-script-id");

    public static void OnLoad()
    {
        _h.UnpatchSelf();                              // prevent patch stacking on hot-reload
        _h.PatchAll(typeof(MyScript).Assembly);
        MelonLogger.Msg("[MyScript] Loaded.");
    }

    public static void OnUnload()
    {
        _h.UnpatchSelf();
        MelonLogger.Msg("[MyScript] Unloaded.");
    }
}
```

`OnLoad` is called after each compile (including hot-reload). `OnUnload` is called before reload or file deletion. Always call `_h.UnpatchSelf()` at the start of `OnLoad` so patches don't stack.

## Disabling a script

Put a bare `off` token at the top of the file (before `using` statements). Roslyn will reject it with a compile error and ScriptEngine silently skips it. Remove the token to re-enable.

## Per-frame code

Scripts don't get `Update()` directly. Attach a `MonoBehaviour` to a persistent `GameObject`:

```csharp
public static void OnLoad()
{
    if (_go != null) GameObject.Destroy(_go);
    _go = new GameObject("__MyScript__");
    GameObject.DontDestroyOnLoad(_go);
    _go.AddComponent<MyBehaviour>();
}
public static void OnUnload()
{
    if (_go != null) { GameObject.Destroy(_go); _go = null; }
}
```

## Harmony patches

Patch classes live at file scope (not nested) alongside the static entry-point class. Use the string overload for private methods:

```csharp
[HarmonyPatch(typeof(Foo), "PrivateMethod")]          // private method by name
[HarmonyPatch(typeof(Bar), nameof(Bar.PublicMethod))] // public method
```

Access private instance fields in Prefix/Postfix parameters using the `____fieldName` convention (4 underscores = private field injection):

```csharp
static void Prefix(ref int ____currentIndex, List<Foo> ____items) { ... }
```

Alternatively use `Traverse` for anything more complex:

```csharp
Traverse.Create(instance).Field("_fieldName").GetValue<T>();
Traverse.Create(instance).Field("_fieldName").SetValue(value);
Traverse.Create(instance).Method("MethodName", new object[] { arg }).GetValue();
```

Use `[ThreadStatic]` reentrancy guards when a Postfix calls the same method it patches:

```csharp
[ThreadStatic] private static bool _resetting;
```

## Logging

Each script writes its own log file to `Scripts/logs/<ScriptName>.log` via `File.AppendAllText`. Use `MelonLogger.Msg` for in-game console output.

## Exploring game types

Decompiled source lives in `decompiled/` (generated by `.\scripts\decompile-game.ps1`). Browse these to find type names, field names, and method signatures. Compile errors like `does not contain a definition for 'X'` mean the name is wrong â€” check the decompiled file. Private fields are prefixed with `_`.

## Key game types (toolbar system)

- **`ToolBarButtonGroup`** â€” owns the list of buttons sharing one shortcut key (`_buttons`, `_hasButtons`, `_currentIndex`). `ActionPerformed` (private) fires on keypress and cycles through non-selected buttons starting from `_currentIndex`.
- **`ToolBarButtonGroupsSO`** â€” ScriptableObject; maps `InputAction â†’ ToolBarButtonGroup` via `_buttonsByInput`. Public `SetLastPressedButton(action, index)` updates `_currentIndex` and calls `SetButtonActiveInGroup` on all buttons in the group.
- **`ToolBarButtonShortcut`** â€” MonoBehaviour on each toolbar button. Holds `_groupInputAction`, `_groupIndex` (its slot in the group), `_isActiveInGroup` (visual highlight). `Init`/`UnInit` add/remove from the group.
- **`SelectObjectToPlaceButton`** â€” concrete `ToolBarButton`; `IsSelected` returns true when its `_factoryObjectData` is the active blueprint in the placement tool.
- **`ToolBarButton.BreadcrumbId`** â€” unique string identifier per button type (e.g. `"MonotonerData"`), used to look up and remap buttons.
